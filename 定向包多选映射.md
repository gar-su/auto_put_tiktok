# 定向包多选映射（TikTok）
## 概述
定向包从全局单选改为按短剧维度多选。用户选定短剧后，通过侧边抽屉为每个短剧分配适用的定向包，引擎按 1:1:n 结构为每个短剧-定向包组合独立创建广告组。支持一键分配以覆盖大部分场景，同时允许个别短剧差异化微调。
## 改动范围
### 移除
- 表单「投放账户」区域下的全局「定向包」单选（原 `selectedTargetingPackage`）
- 账户卡片中的定向包关联入口
### 新增
- 已选短剧区域变更为行布局，每行展示短剧名称 + 已分配定向包标签 +「+ 选择定向包」链接按钮
- 定向包选择侧边抽屉：卡片列表展示定向包（名称 + 定向条件描述），点击卡片切换选中/取消，选中态显示 ✓ 标识
- 抽屉内支持新建、编辑、删除定向包（列表视图 / 表单视图切换），表单字段包括名称、性别、年龄段、语言、操作系统、网络、地区
- 「一键分配定向包」：复用同一侧边抽屉，标题切换为「一键分配定向包」，卡片初始全未选中，抽屉内显示作用域选择器（全部短剧 / 仅未分配的短剧），确定后批量写入
- 「清空定向包」：保留所有短剧，仅清空其定向包分配
- 「清空短剧」：移除所有已选短剧及定向包映射，需二次确认
## 定向包数据模型
TikTok 定向包包含以下定向字段：
| 字段 | 说明 |
|------|------|
| name | 定向包名称 |
| gender | 性别（不限 / 男 / 女） |
| ageGroups | 年龄段（多选：18-24, 25-34, 35-44, 45-54, 55+） |
| languages | 投放语言（多选，使用 ISO 639 二字母代码） |
| os | 操作系统（不限 / iOS / Android） |
| minOsVersion | 最低系统版本（仅限操作系统非不限时） |
| network | 网络环境（不限 / WiFi） |
| locations | 投放地区（逗号分隔的地区名，空字符串表示不限） |

定向包列表为全局共享，所有短剧使用同一套选项。具体定向包名称以实际创建流程中的定向包名为准。
## 短剧选择说明
TikTok 版短剧通过产品文库匹配接入：用户在短剧搜索区输入短剧名称 / 剧本并搜索，在结果中点击「接入产品文库」，通过侧边抽屉为短剧匹配广告账户与产品文库。短剧与账户通过 `dramaMatches` 结构关联，定向包映射独立于产品文库匹配，通过 `dramaTargetingMap` 按短剧 key 存储。
## 交互规则
- 点击短剧行的「+ 选择定向包」打开侧边抽屉，标题「选择定向包」，卡片显示该短剧当前已选状态
- 点击「一键分配定向包」打开同一侧边抽屉，标题「一键分配定向包」，卡片初始全未选中，底部显示作用域选择器
- 点击定向包卡片切换选中/取消，选中卡片有背景色 + ✓ 标识的视觉区分
- 未分配任何定向包的短剧行显示「未分配」
- 一键分配：勾选定向包 → 选择作用域（全部/仅未分配）→ 点击确定，直接覆盖已有分配
- 「仅未分配的短剧」作用域：仅对当前未分配任何定向包的短剧生效
- 移除短剧时自动清除其定向包映射
- 新增短剧时定向包默认为全空（无一选中），须手动分配或使用一键分配
## 提交数据格式
定向包从账户层级下沉到短剧层级：
```json
{
  "dramas": [
    {
      "key": "L0001-繁花-Blooming-英文",
      "targetingPackages": ["tp1", "tp2"]
    },
    {
      "key": "L0001-繁花-Mekar-马来文",
      "targetingPackages": ["tp3"]
    }
  ],
  "account": {
    "package": "账户包A-北美投放"
  }
}
```
`account` 对象中不再包含 `targetingPackage` 字段。
## 投放引擎影响
### 生成规则
引擎遍历 `dramas` 数组，对每个短剧-定向包组合独立创建一条广告系列（campaign），结构保持 **1 campaign : 1 ad set : n ads**。
### 示例
假设提交数据：
- 繁花-英文：`targetingPackages: ["tp1", "tp2"]`
- 繁花-马来文：`targetingPackages: ["tp3"]`
- 广告数 n = 3，计划创建上限 = 5

引擎生成结构：
```
繁花-英文 × 兴趣标签定向
├── Campaign（计划创建上限 = 5）
│   └── Ad Set
│       ├── Ad 1
│       ├── Ad 2
│       └── Ad 3

繁花-英文 × 男性行为定向
├── Campaign（计划创建上限 = 5）
│   └── Ad Set
│       ├── Ad 4
│       ├── Ad 5
│       └── Ad 6

繁花-马来文 × 女性消费定向
├── Campaign（计划创建上限 = 5）
│   └── Ad Set
│       ├── Ad 7
│       ├── Ad 8
│       └── Ad 9
```
- 短剧数 = 2，定向包组合数 = 2 + 1 = 3 个广告组
- 总广告数 = 3 个广告组 × 3 条 = 9 条
- 计划创建上限对各 Campaign **独立生效**，任一 Campaign 内广告数 ≤ 上限时停止创建
### 计算公式
- 广告组数 = Σ(每个短剧的定向包数)
- 理论总广告数 = 广告组数 × n
- 实际总广告数 = Σ min(n, 各 Campaign 的计划创建上限)
## 校验规则
| 条件 | 提示 |
|------|------|
| 短剧列表为空 | 请选择短剧 |
| 任一短剧未分配定向包 | 请为每个短剧分配至少一个定向包 |
| 一键分配时未勾选任何定向包 | 请选择至少一个定向包 |
## 验收标准
1. 删除全局定向包单选组件，账户卡片不再展示定向包关联入口
2. 已选短剧每行展示短剧名称 + 已分配定向包标签 +「+ 选择定向包」链接按钮
3. 点击「+ 选择定向包」打开侧边抽屉，卡片列表展示定向包名称及定向条件描述（性别、年龄、系统、地区）
4. 点击定向包卡片可切换选中/取消，选中态有视觉区分（背景色 + ✓ 标识）
5. 抽屉内支持新建、编辑、删除定向包，列表视图与表单视图可切换
6. 点击「一键分配定向包」打开同一侧边抽屉，标题与作用域选择器区别于单剧模式
7. 一键分配选择「全部短剧」时覆盖所有短剧的定向包分配
8. 一键分配选择「仅未分配的短剧」时仅对未分配短剧生效
9. 「清空定向包」保留所有短剧，仅清空定向包分配，需二次确认
10. 「清空短剧」移除所有短剧及映射，需二次确认
11. 提交时校验每短剧至少一个定向包
12. 提交数据中 `dramas` 数组每项包含 `key` 和 `targetingPackages`
13. 提交数据中 `account` 不含 `targetingPackage` 字段
